# 手动测试清单

## 设置保存测试

1. 打开设置页。
2. 首次安装或清除数据后，分别在系统语言为简体中文、繁体中文、英语和日语时启动 App，确认初始语言跟随系统。
3. 切换简体中文、繁体中文、英语和日语，确认设置页、底部导航、结果页标题和标签文案会切换，重启后仍使用手动选择的语言。
4. 填写 Base URL、模型名称、API Key 和饮食目标。
5. 点击一个内置目标预设，确认饮食目标文本被替换并自动保存，无需点击“保存连接”。
6. 手动编辑饮食目标，确认停止输入后会自动保存。
7. 点击“保存连接”保存模型连接信息。
8. 点击“测试连接”，确认请求会成功并且不提示模型缺少图片输入能力。
9. 关闭并重新打开 App，确认内容仍存在。

## 相册选择测试

1. 首页点击“导入照片”。
2. 选择一张餐食图片。
3. 确认进入分析页并能显示图片。
4. 取消选择时确认不会崩溃。

## 示例图片测试

1. 在没有历史记录时，首页查看“示例图片”横向列表。
2. 点击任意示例图片。
3. 确认进入分析页并能显示该示例图片。
4. 分析成功后确认记录会自动保存到历史。
5. 返回首页后确认示例图片区域隐藏，只展示最近分析记录。

## 拍照测试

1. 首页点击“拍照分析”。
2. 授权相机权限。
3. 拍摄一张餐食图片。
4. 确认进入分析页。

## AI 分析测试

1. 设置页填写可用的 OpenAI-compatible 配置。
2. 使用支持图片输入的模型。
3. 选择或拍摄图片。
4. 确认等待页展示“第 x/4 步”、进度条、分析请求、实时返回和滚动提示。
5. 确认模型尚未输出时只展示友好占位文案，开始输出后再显示“分析中”。
6. 确认等待时可以返回首页，首页能看到后台分析状态，分析完成后记录会自动保存。
7. 确认结果显示餐食名称、怎么吃、1~5 星参考、是否适合、怎么调整、按菜看建议和标签。
8. 使用包含多个菜品、整盘拼盘或一桌菜的图片，确认顶部“怎么吃”、1~5 星参考、“是否适合”和“怎么调整”基于整餐综合判断，而不是只针对某一道菜；如果有更清淡的蔬菜、蛋白或蒸煮类菜品，建议里应提示可以多吃或先吃这些部分。
9. 确认“按菜看建议”仍按菜品展示 2~3 个标签和一句具体、可执行的简短建议。
10. 确认较长的餐食名、摘要、是否适合、怎么调整和按菜看建议会显示“展开”，点击后能显示全文并可收起。
11. 确认标签为短胶囊样式，不出现“常规食材、常规分量、普通、粗估”等低信息量标签。
12. 分别切换到繁体中文、英语和日语后重新分析，确认 AI 返回的新餐食名称、建议、免责声明、标签和按菜建议使用当前语言。
13. 确认首页、历史页和详情页标签优先在同一行利用空间，时间、收藏和操作区域不过度挤占内容。
14. 确认绿色只用于推荐或正向内容，黄色用于需要注意，红色用于明显风险或不推荐内容。
15. 确认页面不展示 kcal、克数或宏量营养素数值，调整建议简短明确，没有大段文字撑破卡片。

## 保存历史测试

1. 分析完成后确认页面显示“已保存到饮食记录”。
2. 点击“回首页”，确认首页最近记录中出现本次分析结果。
3. 打开记录页并点击该记录，确认能进入详情页，无需手动保存。

## 收藏删除测试

1. 在记录页左滑一条记录。
2. 点击“收藏”，再切换“收藏优先”，确认收藏记录靠前。
3. 再次左滑记录并点击“删除”，确认二次确认弹窗出现。
4. 确认删除后列表中消失。

## 错误场景测试

1. 清空 API Key 后分析，确认提示“请先在设置中填写 API Key。”
2. 清空模型名称后分析，确认提示“请先在设置中填写模型名称。”
3. 使用错误 Key，确认提示 Key 可能无效。
4. 使用不支持图片的模型，确认提示模型可能看不了图片。
5. 断网后分析，确认提示请求失败。
