# 手动测试清单

## 设置保存测试

1. 打开设置页。
2. 填写 Base URL、模型名称、API Key 和饮食目标。
3. 点击保存配置。
4. 关闭并重新打开 App，确认内容仍存在。

## 相册选择测试

1. 首页点击“从相册选择”。
2. 选择一张餐食图片。
3. 确认进入分析页并能显示图片。
4. 取消选择时确认不会崩溃。

## 示例图片测试

1. 首页查看“示例图片”横向列表。
2. 点击任意示例图片。
3. 确认进入分析页并能显示该示例图片。
4. 分析成功后确认记录会自动保存到历史。

## 拍照测试

1. 首页点击“拍照分析”。
2. 授权相机权限。
3. 拍摄一张餐食图片。
4. 确认进入分析页。

## AI 分析测试

1. 设置页填写可用的 OpenAI-compatible 配置。
2. 使用支持图片输入的模型。
3. 选择或拍摄图片。
4. 确认显示餐食名称、热量、宏量营养素、目标匹配、建议、食材和标签。
5. 使用包含多个菜品或复合食材的图片，确认“多食材明细”能展示主要菜品和食材。
6. 确认标签为短胶囊样式，建议文案简短明确，没有大段文字撑破卡片。

## 保存历史测试

1. 分析完成后确认页面显示“已自动保存到记录”。
2. 点击查看记录，确认能进入详情页。
3. 打开记录页，确认记录存在，无需手动保存。

## 收藏删除测试

1. 在记录页点击收藏按钮。
2. 切换“收藏优先”，确认收藏记录靠前。
3. 删除一条记录，确认列表中消失。

## 错误场景测试

1. 清空 API Key 后分析，确认提示“请先在设置中配置 API Key。”
2. 清空模型名称后分析，确认提示“请先在设置中填写模型名称。”
3. 使用错误 Key，确认提示 Key 可能无效。
4. 使用不支持图片的模型，确认提示模型可能不支持图片分析。
5. 断网后分析，确认提示网络请求失败。
