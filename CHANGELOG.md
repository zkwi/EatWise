# 变更记录

本项目采用轻量变更记录，面向个人维护和后续 AI 接手。版本号在正式发布 APK 时再统一整理。

## 1.0.6 - 2026-05-11

- 精修结果页“怎么调整”行布局，动作标签固定宽度并与右侧建议文案垂直居中对齐。
- 精修“按菜看建议”行布局，菜品名称列和标签列保持稳定起点，标签文字居中显示。
- 菜品名称列按当前语言和标题内容自适应，避免历史记录里中英文混排菜名过早截断。
- 补充结果页布局规则单元测试，覆盖建议标签宽度、行内垂直对齐、菜名列宽和按菜标签最小宽度。

## 1.0.5 - 2026-05-11

- 按菜建议改为更丰富的多标签展示，并压缩行距和卡片内边距，减少结果页留白。
- 丰富结果摘要、适配理由和调整建议文案，让每条建议更接近可执行的吃法提醒。
- 优化 prompt 到 v12，要求模型按菜识别烹调方式、主食/蛋白/蔬菜线索，并避免低信息量标签。
- 修正 `sweet potato` 被误判为高糖的按菜标签问题。
- 分析页遇到 API Key、模型名或图片模型配置错误时，直接提供“设置”入口，减少失败后的死路。
- 放宽英文紧凑标签宽度，修正首页和历史记录卡片中 `Multiple dishes`、`Enough protein` 等标签截断。
- 在模拟器复查首页、历史、详情、设置、相机权限/拍照、无配置失败态和系统照片导入入口。

## 1.0.4 - 2026-05-11

- 设置页饮食目标支持选择或编辑后自动保存，预设和自定义目标不再依赖手动保存连接设置。
- 结果卡片中的长文本支持展开和收起，避免直接省略为三个点。
- README 增加基于内置测试图片和合成记录生成的主页面截图。
- 统一 App 和文档中的普通用户文案，结果卡标题改为“是否适合/怎么调整/按菜看建议”。
- 精修结果页建议项和列表标签展示，减少单行截断，让建议更像可执行清单。
- 强化 prompt 和建议清洗规则，避免输出抽象提醒，优先生成少蘸酱、少喝汤、主食少几口等具体建议。
- 新增简体中文、繁体中文、英语和日语语言设置，新分析的 prompt、建议、标签和免责声明会按当前语言输出。
- 优化多语言翻译和窄屏排版，修正英文/日文标签、繁体中文术语和长按钮文本截断。
- 首次启动会按系统默认语言选择初始语言，并补齐大模型提示词、建议和标签的多语言别名归一化。
- 精修多语言审计细节，按菜建议的本地提示可识别英文和日文食材关键词，降低非中文结果退回普通提示的概率。

## 1.0.3 - 2026-05-11

- 等待页增加分析请求和实时返回预览，让长时间分析过程更透明。
- 优化首页、历史页和详情页的标签布局，压缩无效留白并展示更多关键信息。
- 记录卡继续收紧时间和收藏区域占比，标签可在同一行展示更多有效判断。
- 详情页主建议区域前移短标签，减少重复模块并提升移动端首屏利用率。
- 同步文档、手动测试清单和维护说明，为流式输出与真机截图验收补充说明。

## 1.0.2 - 2026-05-11

- 增加图片分析阶段进度、滚动提示和后台分析入口，降低长时间等待的不确定感。
- 图片上传前限制最大宽高并保持清晰度优先，减少手机原图带来的请求开销。
- 首页、历史页、设置页和详情页继续收紧移动端排版，提升首屏信息密度。
- 历史记录改为左滑收藏或删除，普通列表不再常驻操作按钮。
- 结果页增加 1~5 星参考，按菜展示多个标签和简短调整建议。
- 同步 AI 治理、维护说明和手动测试清单，并更新 prompt 版本。

## 1.0.1 - 2026-05-10

- 将分析结果改为是否适合目标和怎么吃，不再展示不可靠的卡路里、重量和宏量营养素估算。
- 替换应用图标，统一底部导航、按钮、卡片、标签和列表的视觉风格。
- 设置页测试连接会发送内置图片，验证模型支持多模态输入。
- 首页示例图片仅在没有历史记录时展示，避免用户已经导入图片后继续被示例引导打扰。
- 增加 MIT License，并补充发布前校验、隐私边界和手动测试说明。

## 1.0.0 - 2026-05-10

- 优化分析结果标签和建议文案，使其更短、更生活化、更适合手机卡片展示。
- 精修首页、分析页、相机页、历史页、详情页和设置页的交互文案。
- 为历史页和详情页删除操作增加二次确认。
- 收敛默认模型配置和默认饮食目标，减少重复字符串带来的维护成本。
- 增加贡献指南、行为准则、安全策略、Issue/PR 模板和更完整的忽略规则。
- 关闭 Android 系统备份并显式排除本地数据，降低 API Key、图片和分析记录泄露风险。
- 增加 release 签名配置和发布说明，签名密钥与密码仅从本地配置读取。
- 同步 README、维护说明和手动测试清单。
